What even is ST0N3?

ST0N3 is a newly-discovered exploit that can unenroll an enterprise enrolled Chromebook. This exploit uses RMA shims, which is only intended for use by manufacturers and repair shops, but were leaked. By using this exploit, we can boot into developer mode, enter recovery and boot off of the shim USB like it was a recovery image. Since Google left a rootfs vulnerability unpatched, we were able to run unsigned code; and therefore unenrollment.

Okay, so, how did you find ST0N3?

Even though Google has patched unenrollment, the new exploit was discovered by a group of ethical hackers during a routine security audit of school Chromebooks. The hackers identified a vulnerability in the update-engine that could be exploited to remove enterprise enrollment, leading to the creation of the exploit. The exploit uses RMA shims to manipulate update-engine to allow for unenrollment when rebooting back into ChromeOS.
